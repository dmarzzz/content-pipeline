# aesthetic

Pluggable style layer for the pipeline. Each format's prompt reads
whichever aesthetic file is active in `config.yaml → active_aesthetic`.

## Default

[`default.yaml`](./default.yaml) is shipped with the pipeline. It's
a deliberately mild house style that any user can read and critique.
Do not edit this file to fit your personal voice; it's the baseline
that new users inherit.

## Make it yours

Copy the default and name the copy after yourself or your project:

```bash
cp aesthetic/default.yaml aesthetic/your-handle.yaml
# edit aesthetic/your-handle.yaml to reflect your voice
```

Then in `config.yaml`, set:

```yaml
active_aesthetic: aesthetic/your-handle.yaml
```

Everything except `default.yaml` and this README is **gitignored**, so
you can iterate on your personal style without it leaking into a PR
by accident.

## Fields that matter

- `voice.forbidden` and `voice.encouraged` are the strongest signals
  to the writing agent. Be specific; generic guidance does nothing.
- `format_specific.<format>.*` overrides per-format knobs (length
  target, opening style, whether to pair a tweet thread with an
  explainer video, etc).
- `visual_identity.*` drives the HTML explainer video's CSS and any
  other visual output.

## Iterating

An agent helping you refine your aesthetic should:

1. Read one or more of your existing published pieces.
2. Compare them to what it just generated under the current aesthetic.
3. Propose specific edits to `voice.forbidden` / `voice.encouraged` /
   `sentence_rhythm` / etc. to close the gap.
4. Leave `default.yaml` alone.
